# Target Typology v1

## Purpose

This protocol defines the target classes used by the system.

## Type A — Operator Under Constraint
Characteristics:
- real system exposure
- active failure pressure
- ownership present

Default route:
- conversion path

## Type B — Architectural Peer
Characteristics:
- high conceptual fluency
- low direct ownership pressure
- useful for alignment, model refinement, or collaboration

Default route:
- alignment path

## Type B-Strategic — Adjacent Founder
Characteristics:
- already selling adjacent solution
- strong narrative control
- useful for asymmetry-based engagement

Default route:
- destabilization path

## Type C — Constraint-Aware but Incomplete
Characteristics:
- senses failure
- lacks structure
- potentially convertible after shaping

Default route:
- shaping path

## Type D — Curious Observer
Characteristics:
- interested but not materially exposed

Default route:
- filter

## Type E — Noise
Characteristics:
- low relevance
- low signal
- no ownership

Default route:
- drop
