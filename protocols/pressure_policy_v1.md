# Pressure Policy v1

## Purpose

This protocol defines the maximum pressure allowed by target type.

## Principle

Pressure should increase precision, not aggression.

## Target Pressure Ceilings

### Type A
Pressure ceiling:
- high, but narrow

Use:
- concrete execution paths
- authority/evaluation mismatch
- interrupt failure

Avoid:
- abstraction surplus

### Type B
Pressure ceiling:
- medium

Use:
- one missing invariant
- one deeper failure mode
- alignment through technical precision

Avoid:
- direct selling

### Type B-Strategic
Pressure ceiling:
- medium, low-visibility

Use:
- subtle destabilization
- missing runtime layer
- product-narrative asymmetry

Avoid:
- overt challenge posture

### Type C
Pressure ceiling:
- low to medium

Use:
- structure
- failure naming
- one mechanism at a time

Avoid:
- premature conversion

### Type D
Pressure ceiling:
- low

Use:
- minimal response

### Type E
Pressure ceiling:
- none

Use:
- no engagement

## De-escalation Conditions

Reduce pressure when:
- signal drops
- target becomes generic
- topic drifts
- resistance repeats without new structure
